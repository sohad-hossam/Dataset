Using @LoadAsync with parameter for code partition can cause problems with errai ui
This bug is related to https://issues.jboss.org/browse/ERRAI-703.

Using the new parameter added to @LoadAsync in the context of errai ui can cause problems.

Using a setup like the 2 classes below produces an error in TemplateUtil.java:75 when accessing field.getElement().

@Templated("TestTemplate.html")
@LoadAsync(ParentWidget.class)
public class ParentWidget extends Composite {
@Inject
@DataField("content") 
private ChildContentWidget w;
public Widget getW()
{ return w; }
}
@Templated("TestTemplate.html#content")
@LoadAsync(ChildContentWidget.class)
public class ChildContentWidget extends Composite {
}
