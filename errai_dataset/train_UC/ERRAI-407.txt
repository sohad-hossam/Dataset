Creating an @EventHandler without appropriate method arguments causes ArrayIndexOutOfBoundsException
If done correctly, this works fine:

{code}

	@EventHandler("button")
	public void buttonHandler(ClickHandler event) {
		this.removeFromParent();
	}
{code}

The bad code does not, note the missing event type in the method signature:

{code}

	@EventHandler("button")
	public void buttonHandler(/*missing event type*/) {
		this.removeFromParent();
	}
{code}
